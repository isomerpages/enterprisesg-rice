<p><strong>CHANGES TO RICE PERMIT DECLARATION PROCEDURES</strong>
  <br />
  &nbsp;
</p>
<p>1.&nbsp;&nbsp;&nbsp; The following changes to rice permit declaration procedures will&nbsp;<strong><em>take effect on 7 January 2013</em></strong>.
</p>
<p>&nbsp;&lsquo;Product Code&rsquo; to be included as a mandatory field
</p>
<p>&nbsp;2.&nbsp;&nbsp;&nbsp; In addition to HS Code, Product Code will be included as a mandatory field to further segregate the rice types imported. All importers will be required to state the Product Code of the type of rice to be imported in the field &lsquo;<em>Product Code&rsquo;</em>&nbsp;during permit declaration. Please refer to Annex A for the list of rice types and their respective Product Codes.
</p>
<p>&nbsp;Descriptions on Packing of Rice no longer required
</p>
<p>&nbsp;3.&nbsp;&nbsp;&nbsp; Declaration of permit no longer requires inputs on the rice packing descriptions or the cost &amp; exchange rate information at the fields &lsquo;Item Description&rsquo;/ &lsquo;Declaration Remarks&rsquo;.
</p>
<p>&nbsp;Purpose of import and the specific month for that purpose or reference number to be indicated at &lsquo;CA/SC Code 1&rsquo; and &lsquo;CA/SC Code 2&rsquo; respectively
</p>
<p>&nbsp;4.&nbsp;&nbsp;&nbsp; You are required to type the &lsquo;purpose of import&rsquo; (e.g. MIQ, PSQ, re-export, conversion of MIQ) in the field&nbsp;<em>&lsquo;CA/SC Code 1&rsquo;&nbsp;</em>and the&nbsp;<em>&lsquo;</em>specific month it is intended for&rsquo; or &lsquo;reference number&rsquo;<em>&nbsp;in the field &lsquo;CA/SC Code 2&rsquo;</em>. You may refer to Annex B as a guide on how to input these two mentioned fields.
</p>
<p>&nbsp;
</p>
<p><strong>AUTOMATION OF RICE PERMIT PROCESSING</strong>
</p>
<p>&nbsp;5.&nbsp;&nbsp;&nbsp; In a move to provide faster turnaround time for rice permit processing, IE Singapore will be rolling out automated permit processing (for both import and export permits)&nbsp;in two phases. Unless picked by the system for random check, all permits will be processed by the system and the outcome would be made known instantly and supporting document is no longer required.
</p>
<p>&nbsp;6.&nbsp;&nbsp;&nbsp; If a permit is picked for random check, the system will prompt the importer to attach the supporting document and it will be routed for manual processing. The turn-around time for manually-processed permits will remain as one business day. If a permit is found to contain mistake(s) during random check, the rice licensee (regardless of who, the importer himself or the declaring agent, declared the permit) will be suspended from automated permit processing for a period of one month. During this suspension period,&nbsp;<strong>ALL</strong>&nbsp;permits will be routed for manual processing. The suspension period will restart if there is any subsequent mistake found in declaration.&nbsp;
</p>
<p>&nbsp;7.&nbsp;&nbsp;&nbsp; A permit may also be automatically rejected by generic TradeNet declaration criteria which include the following:
</p>
<ol>
  <li>The declared CA/SC product quantity does NOT match that stored in the product database (including the unit used)</li>
  <li>UEN does NOT match License number.</li>
  <li>Invalid or terminated license number used.</li>
</ol>
<p>The rice licensee will also be subjected to suspension from automatic processing (for a period of one month) by the system if such mistake is made during declaration.
</p>
<p>8.&nbsp;&nbsp;&nbsp; To qualify for automatic permit processing, the rice licensee has to fulfill&nbsp;<strong><em>BOTH</em></strong>&nbsp;the following criteria:
</p>
<ol>
  <li>Fulfilling Stockpile Quantity requirement at the point of permit declaration&nbsp;<em>(applicable to&nbsp;all&nbsp;stockpile grade rice importers only)</em></li>
  <li>Not under suspension from automatic permit processing</li>
</ol>
<p>9.&nbsp;&nbsp;&nbsp;&nbsp;The following types of permits will be automatically processed under Phase 1, which will commence&nbsp;on 7 Jan 2013:
</p>
<ol>
  <li>Import permits of non-stockpile grade rice</li>
  <li>Import permits of white rice</li>
  <li>Import permits of rice for manufacturing</li>
  <li>Re-export/ export permits of all types of rice</li>
</ol>
<p>10. The following types of permits will be included in&nbsp;Phase 2&nbsp;(date of roll-out will be announced at a later stage):
</p>
<ol>
  <li>Import permits of Parboiled rice, Ponni rice and Basmati rice.</li>
</ol>

</p>

<p><strong>Frequently Asked Questions</strong>
</p>
<p>
  <br />
  &nbsp;
</p>
<p>Q1.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; What is the turn-around time for Auto-processing?
</p>
<p>A1:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Immediate.
</p>
<p>&nbsp;
</p>
<p>Q2:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; What is the processing time for permits picked for random checks and those manually processed?
</p>
<p>A2:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; One business day from the time supporting document is submitted.
</p>
<p>&nbsp;
</p>
<p>Q3:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; When will Phase 2 be launched?
</p>
<p>A3:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; All importers will be duly informed once the date of launch for Phase 2 is decided.
</p>
<p>&nbsp;
</p>
<p>Q4:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; If I declare MIQ for a month, say Jan 13, and I exceed the MIQ by a small quantity, will the system immediately reject my permit?
</p>
<p>A4:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; No, it will not. At our end, we will adjust the excess to the MIQ for the month after.
</p>
<p>&nbsp;
</p>
<p>Q5:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Will the system reject if I declare the import quantity with too many decimal places? How many decimal places are permissible?
</p>
<p>A5:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; No, the system will not. However, importers are advised to declare up to two decimal places.
</p>
<p>&nbsp;
</p>
<p>Q6: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Is cancellation of permits processed manually?
</p>
<p>A6:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The procedure for permit cancellation remains unchanged.
</p>
<p>&nbsp;
</p>
<p>Q7:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; When prompted to attach supporting documents, how do I attach? &nbsp;
</p>
<p>A7:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Login to the TradeNet BackEnd website using FrontEnd User ID and password. Search for the Declaration Unique Reference Number. Add the supporting document and confirm the upload.
</p>
<p>&nbsp;
</p>
<p>Q8:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; What should re-exporters input in the fields &lsquo;CA/SC Code 1&rsquo; and &lsquo;CA/SC Code 2&rsquo;?
</p>
<p>A8:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Re-exporters do not need to input anything in fields &lsquo;CA/SC Code 1&rsquo; and &lsquo;CA/SC Code 2&rsquo;.
</p>
<p>&nbsp;
</p>
<p>Q9:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Do manufacturing licensees need to indicate MIQ in &lsquo;CA/SC Code 1&rsquo; and &lsquo;CA/SC Code 2&rsquo;?

<p>A9:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Yes, manufacturing licensees need to indicate MIQ in &lsquo;CA/SC Code 1&rsquo; and the respective MIQ month and year in &lsquo;CA/SC Code 2&rsquo;.
